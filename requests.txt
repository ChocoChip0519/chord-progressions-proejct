[STACK] 기술 스택
--------------------------------------------------------------------------------
  - 프론트엔드 : React + TypeScript
  - 백엔드/DB  : Supabase (PostgreSQL)
                 * chord_edges 테이블 : 코드 간 전이 데이터 (from, to, genre, weight)
                 * cards 테이블       : 저장된 코드 진행 카드
                 * folders 테이블     : 사용자 폴더
                 * card_folders 테이블: 카드-폴더 다대다 관계
  - 음원 재생  : Tone.js (Web Audio API 기반 피아노 음 합성)
  - 드래그앤드롭: dnd-kit

  [자료구조 — TypeScript로 직접 구현]
  - Graph (인접 리스트)  : 코드 추천 엔진 핵심
                           Supabase chord_edges 데이터를 로드하여 그래프 구성
                           현재 코드 기준 엣지 탐색 → 가중치 내림차순 정렬 → 추천 반환
  - Stack               : 코드 진행 히스토리 관리
                           코드 확정 시 push / Backspace 시 pop
  - Queue               : 추천 카드 슬라이드 관리
                           추천 결과를 큐에 담아 좌우 순환 탐색
  - Tree                : 폴더 구조 관리
                           폴더 생성/삭제/탐색을 트리로 관리

[HOME] 홈 화면 (최초 접속 시)
--------------------------------------------------------------------------------
  1. 레이아웃
     - 상단: 폴더 탭 바  →  작업 list | list1 | list2 | +
     - 탭 클릭 시 해당 폴더의 카드 그리드로 전환
     - 그리드 가로 4열 기준:
         x  x  x  +    ← 첫 줄 마지막 셀이 '새 코드 진행 만들기' 버튼 (그리드 인라인 고정)
         x  x  x  x
         x  x  x  x  ...
     - 세로 스크롤로 전체 목록 탐색 가능

  2. 폴더 구조
     - [작업 list] : 저장된 모든 코드 진행이 기본으로 모이는 곳 (전체 보관함)
     - [list1, list2, ...] : 사용자가 직접 만드는 폴더
       * 작업 list의 카드를 원하는 폴더에 추가 가능 (카드 내 폴더 지정 버튼 or 드래그앤드롭)
       * 하나의 카드를 여러 폴더에 동시에 넣을 수 있음
       * 폴더 탭 우측 + 버튼으로 새 폴더 생성 / 폴더명 클릭하여 이름 수정 / 폴더 삭제 가능
         (폴더 삭제 시 카드는 삭제되지 않고 작업 list에 유지)

  3. 드래그앤드롭 인터랙션
     - [폴더 간 이동] 카드를 드래그하여 상단 폴더 탭 위에 올리면 해당 폴더에 추가
     - [카드 위치 재배열] 휴대폰 앱 정렬 방식과 동일
       * 카드를 길게 누르거나(long press) 드래그 시작 시 편집 모드 진입
       * 편집 모드에서 카드가 살짝 떠오르며 그림자가 생기는 float 형태로 전환
       * 카드를 드래그하여 원하는 위치로 이동, 다른 카드들이 자동으로 밀려남
       * 빈 영역 클릭 또는 완료 버튼으로 편집 모드 종료
       * 재배열된 순서는 저장됨 (정렬 기준을 '사용자 지정'으로 자동 전환)

  4. 카드(x) 구성
     - 코드 진행 제목
     - 저장된 코드 진행 요약 (예: C - Am - F - G)
     - 장르 태그, 저장 일시 등 간략 정보
     - 폴더 지정 버튼 : 이 카드를 넣을 폴더 선택
     - 🗑 버튼 : 카드 삭제 (삭제 확인 모달 표시 후 삭제)

  5. 정렬 / 필터 (폴더별 독립 적용)
     - 제목 이름순 정렬
     - 최신순 정렬

[EDITOR] 코드 작업 화면 (홈에서 + 또는 카드 클릭 시 진입)
--------------------------------------------------------------------------------
  0. 진입 흐름
     - 홈 화면에서 + 클릭 시 새 코드 작업 페이지로 이동
     - 진입 시 순서:
       1) 제목 입력 (빈칸 가능, 나중에 수정 가능)
       2) 장르 선택
       → 설정 완료 후 코드 작업 화면으로 전환
     - BPM은 코드 작업 화면 진입 후 언제든 수정 가능
     - 작업 중 저장 시 홈 화면 작업 list에 카드로 추가됨

  1. 장르 선택
     - 사용자가 원하는 음악 장르를 선택 (예: Pop, Jazz, Rock, R&B 등)

  2. BPM 설정
     - 원하는 BPM을 숫자로 직접 입력 (범위: 40 ~ 300, 기본값: 120)

  3. 피아노 건반 인터랙션
     - 화면에 피아노 건반 UI 표시
     - 키보드 키와 건반 1:1 매핑 (예: A~K → 흰 건반, W/E/T/Y/U → 검은 건반)
     - 건반 위에 해당 키보드 키 힌트만 표기 (음 이름 표기 없음)
     - 건반 클릭 또는 키보드 입력으로 음 재생
     - 여러 음을 동시에 누르면 구성음을 분석하여 코드명으로 인식
       (예: C+E+G 동시 입력 → "C" 로 인식)

  4. 코드 입력 및 확정
     - Enter  : 현재 누른 음들을 코드명으로 분석하여 확정
               (표시 형식: "A+D+G" 가 아닌 "C" 같은 코드명으로 표시)
     - Backspace : 마지막 확정 코드 삭제
     - → (오른쪽 화살표) : 다음 코드 추천 받기

  5. 코드 진행 히스토리
     - 확정된 코드 진행을 순서대로 건반 UI 위쪽에 표시
     - 새 코드 확정 시 오른쪽으로 추가됨
     - Backspace 시 가장 마지막 코드 제거

  6. 몰입형 코드 작성 팝업 (→ 입력 시 진입)
     - → 화살표 입력 시 현재 화면 위에 오버레이 팝업으로 전환 (풀스크린 or 화면 대부분 점유)
     - ESC 키 입력 시 팝업 닫고 메인 화면으로 복귀
     - 팝업 내부 구성 (위→아래):

     [영역 1] 확정 코드 진행 히스토리
     - 지금까지 확정된 코드를 순서대로 표시
     - 메인 화면의 히스토리와 동일하게 실시간 반영

     [영역 2] 추천 코드 float 카드
     - 현재 코드 진행을 기반으로 다음 코드 후보 3~4개 추천
       (예: Am7, Am9, AmMaj7 ... 같은 변형 포함)
     - 사전 데이터 기반 추천 (규칙/점수제)
       * 장르별 유명 코드 진행 패턴을 사전에 DB화
       * 현재 코드 진행과 패턴 매칭 후 빈도/가중치 점수로 정렬
       * 점수 높은 순으로 카드 앞쪽(중앙)에 노출
       * 지원 장르 (4개): Pop, R&B, 발라드, Rock — 장르별 데이터 각각 구축
     - 드물게 쓰이는 코드 → 좌/우 화살표로 슬라이드하여 탐색

     [카드 구성 — 방식 A] 클릭 즉시 재생
     - 카드에 코드명 표시
     - 카드 클릭 시 해당 코드의 구성음이 동시에 자동 재생

     [카드 구성 — 방식 B] 직접 연주
     - 카드에 코드명 + 미니 건반 (구성음 하이라이트 + 키보드 키 표기)
     - 사용자가 직접 키보드 or 클릭으로 구성음을 눌러볼 수 있음
     - 미니 건반이 실제 눌린 상태로 시각적으로 반응

     - 방식 A, B 둘 다 구현
     - 마음에 드는 카드 선택(클릭 or Enter) 시 해당 코드 확정 후 히스토리에 추가

     [영역 3] 피아노 건반
     - 팝업 최하단에 메인 건반과 동일한 건반 UI 배치
     - 직접 음을 눌러 새 코드를 조합하고 확정 가능
     - 팝업 안에서 메인 화면과 동일하게 코드 입력/확정 흐름 유지

[DESIGN] 디자인 요소
--------------------------------------------------------------------------------
  1. 전체 레이아웃 (위→아래 순서)
     - 최상단 : 이번 코드진행의 제목 (클릭하여 언제든 수정 가능)
     - 그 아래 : 장르 / 비트 설정 정보
     - 그 아래 : 확정된 코드 진행 히스토리
     - 최하단 : 피아노 건반 UI

  2. 몰입형 팝업 오버레이 (→ 입력 시)
     - 현재 화면 위에 오버레이로 표시 (풀스크린 or 화면 대부분 점유)
     - 배경은 반투명 처리하여 몰입감 강화
     - ESC 입력 시 팝업 닫힘

     [팝업 내부 레이아웃 — 위→아래]
     - 상단 : 확정 코드 진행 히스토리 (메인과 동일하게 실시간 반영)
     - 중단 : 추천 코드 카드 3~4개 float 표시
              * 각 카드: 코드명 + 미니 건반 (구성음 하이라이트 + 키보드 키 표기)
              * hover 또는 클릭 시 해당 코드 소리 재생
              * 선택된 카드는 강조 표시
              * 좌/우 슬라이드로 추가 추천 카드 탐색 가능
     - 하단 : 피아노 건반 UI (메인 건반과 동일, 코드 입력/확정 가능)

  3. 건반 하이라이트 색상 구분 (메인 건반 + 미니 건반 공통)
     - 추천 구성음     : 파란색 계열
     - 사용자가 누른 음 : 별도 색상 구분
     - 일반 건반      : 기본 흰/검은 건반 색상
